## 오늘 새로 준비하는 것 2개

### 1. GitHub — 드디어 씁니다

1회차엔 계정만 만들었죠. 오늘부터 진짜 씁니다.

**왜 필요한가**
내일 Vercel이 **GitHub에서 코드를 가져가서** 배포합니다. 여기 코드가 없으면 배포를 못 해요.

**용어 세 개**

| 말 | 뜻 |
|---|---|
| **저장소 (repo)** | 프로젝트 하나가 통째로 들어가는 폴더 |
| **커밋 (commit)** | "여기까지 작업한 걸 저장" |
| **푸시 (push)** | 내 노트북에 저장한 걸 GitHub로 올리기 |

**어떻게 하나** — AI에게 시킵니다.

```
이 프로젝트를 GitHub에 올려줘.
저장소 이름은 my-site 로, 비공개로 만들어줘.
.env 파일은 올라가지 않게 확인해줘.
```

> **`.env` 는 절대 올라가면 안 됩니다.** Supabase 키가 들어 있으니까요.
> AI가 알아서 해주지만, 위 프롬프트에 한 번 더 말해두는 게 안전합니다.

## 오늘 하는 것 3단계

**4회차에서 못 끝낸 사람 먼저입니다.**

- Supabase 연결 안 된 사람 → 오늘 뚫습니다

**이미 다 된 사람**은 이 시간에 다듬기를 먼저 시작하세요. 그리고 옆 사람을 도와주세요.

**오늘 안에 6명 다 "동작하는 사이트"까지 갑니다.**

---

### 2단계 · 완성 체크리스트 (30분)

여섯 개를 하나씩 확인합니다.

- [ ] **기능 하나는 실제로 동작한다**
- [ ] **화면에 이미지가 하나 이상 있다**
- [ ] **브라우저 탭에 사이트 이름이 뜬다**
- [ ] **텍스트에 오타가 없다**
- [ ] **색과 글씨가 최소한 통일돼 있다**
- [ ] **휴대폰 화면에서도 안 깨진다**

> **6개 다 못 해도 됩니다. 4개 이상이면 내일 배포 가능합니다.**

**프롬프트 모음**

```
지금 화면에서 이 부분을 고치고 싶어:
- 제목 글씨가 너무 작아. 두 배 정도 크게
- 카드 사이 간격이 좁아. 조금 띄워줘
- 배경이 흰색인데 아주 옅은 회색으로

한 번에 하나씩 하고, 하고 나면 알려줘.
```

```
브라우저 탭에 뜨는 이름을 [내 카페 기록] 으로 바꿔줘.
```

```
휴대폰 화면에서 봤을 때도 안 깨지게 해줘.
```

### 3단계 · 옆 사람에게 노트북 넘기기 (20분)

**오늘의 진짜 순서입니다.**

2인 1조로 앉아서, **자기 노트북을 옆 사람에게 넘깁니다.**

- **만지는 사람:** 처음 온 사용자처럼 자연스럽게 눌러보세요
- **만든 사람:** 개입하지 마세요. **보기만 하세요**

3분씩 하고 짝을 바꿉니다.

**여기서 이런 말이 나옵니다.**
> "여기가 뭐예요?"
> "이거 눌렀는데 왜 안 되죠?"

**그게 오늘 제일 중요한 발견입니다.**

내 눈엔 당연했던 게 처음 오는 사람 눈엔 안 보여요. 배포 전에 발견해서 다행인 겁니다.

> **상처받지 마세요.** 옆 사람이 못 알아본 건 여러분이 잘못한 게 아닙니다.
> 처음 오는 사용자 눈으로 보면 다들 그렇게 보여요.
