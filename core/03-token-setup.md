# 토큰 절약 설정 (하네스 세팅)

> **읽는 시점:** 1회차 직후 · 한 번 세팅하고 잊습니다
> **읽는 시간:** 15분 · 세팅 시간 10분
> **효과:** 6주 동안 AI 사용량이 절반 이하로 줄어듭니다

---

## 왜 이걸 해야 하나요

### 문제 1 · 대화가 길어질수록 비싸집니다

AI는 여러분과 나눈 대화를 **매번 처음부터 다시 읽습니다.**

```
1번째 메시지  →  AI가 읽는 양: 1개
10번째 메시지 →  AI가 읽는 양: 10개 전부
50번째 메시지 →  AI가 읽는 양: 50개 전부
```

그래서 2시간짜리 회차의 **마지막 30분이 앞의 90분보다 더 많이 씁니다.**
유료 구독에도 사용량 한도가 있어요. 회차 중간에 한도가 차면 남은 시간을 못 씁니다.

### 문제 2 · 대화가 길어질수록 AI가 멍청해집니다

이게 더 중요합니다.

AI가 한 번에 볼 수 있는 양(컨텍스트 창)은 정해져 있어요. 그게 **절반쯤 차면 이미 품질이 떨어지기 시작합니다.** 꽉 차야 문제가 생기는 게 아니에요.

- 아까 정한 규칙을 잊습니다
- 앞에서 만든 파일을 또 만듭니다
- 엉뚱한 파일을 고칩니다

**"AI가 점점 이상해지네"** 싶으면 대부분 이 문제입니다.

---

## 해결책은 두 개뿐

| | 하는 것 | 언제 |
|---|---|---|
| **1. CLAUDE.md 파일** | 매번 설명할 걸 파일에 적어둔다 | 1회차에 한 번 만들고 계속 수정 |
| **2. 슬래시 명령어 3개** | 대화가 길어지면 정리한다 | 회차 중간중간 |

---

# 1. CLAUDE.md 만들기

## 이게 뭔가요

**AI에게 매번 설명하기 귀찮은 것을 적어두는 파일**입니다.

프로젝트 폴더 맨 위에 `CLAUDE.md` 라는 파일을 만들어두면, Claude Code가 **대화를 시작할 때마다 자동으로 읽습니다.**

그래서 매번 이렇게 안 말해도 됩니다.
> "한국어로 대답해줘. 이건 Next.js 프로젝트고, Supabase 쓰고, 파일 고치기 전에 뭘 바꿀지 먼저 말해줘..."

한 번 적어두면 끝.

> **참고:** Codex를 쓰는 분은 `AGENTS.md` 라는 이름으로 만듭니다. 내용은 똑같습니다.

---

## 만드는 법

프로젝트 폴더에서 Claude Code를 켜고 이렇게 칩니다.

```
/init
```

AI가 프로젝트를 훑어보고 `CLAUDE.md` 초안을 만들어줍니다.
그다음 아래 템플릿을 참고해서 고치세요.

---

## 우리 커뮤니티용 템플릿

**그대로 복사해서 대괄호만 바꾸세요.**

```markdown
# 프로젝트

[브런치 카페 리뷰를 기록하는 개인 사이트]

## 스택

- Next.js (App Router)
- Supabase (데이터 저장)
- Vercel (배포)

## 명령어

- 개발 서버 켜기: `npm run dev`
- 빌드: `npm run build`

## 나에 대해

- 나는 코딩을 모른다. 코드를 직접 쓰지 않는다.
- 전문 용어를 쓸 거면 괄호 안에 쉬운 말로 풀어써라.

## 작업 규칙

- 한국어로 답한다.
- 설명은 짧게. 세 줄을 넘기지 않는다.
- 파일을 고치기 전에, 무엇을 왜 바꾸는지 한 줄로 먼저 말한다.
- 한 번에 파일 하나씩만 고친다.
- 새 라이브러리를 설치하기 전에 반드시 물어본다.
- 내가 요청하지 않은 파일은 건드리지 않는다.

## 하지 말 것

- `.env` 파일의 내용을 화면에 출력하지 않는다.
- API 키를 코드 안에 직접 적지 않는다. 반드시 `.env` 를 쓴다.
- 내가 이해하지 못할 것 같은 복잡한 구조를 만들지 않는다.

## 페이지 구조

(2회차에 정한 페이지 목록을 여기 붙여넣기)

## 데이터 구조

(4회차에 만든 Supabase 테이블을 여기 적기)
```

---

## 회차마다 이렇게 자랍니다

| 회차 | CLAUDE.md에 추가할 것 |
|---|---|
| **1회차** | 위 템플릿 그대로 (페이지·데이터 구조는 비워둠) |
| **2회차** | 정한 페이지 목록을 `## 페이지 구조` 에 붙여넣기 |
| **4회차** | Supabase 테이블 이름과 칸 이름을 `## 데이터 구조` 에 적기 |
| **아무 때나** | AI가 자꾸 같은 실수를 하면 → `## 작업 규칙` 에 한 줄 추가 |

**마지막 줄이 핵심입니다.**
"파일을 너무 많이 고치네" 싶으면 그때그때 규칙을 한 줄 추가하세요. 그게 이 파일의 목적입니다.

---

## 주의: 너무 길게 쓰지 마세요

CLAUDE.md도 결국 AI가 매번 읽는 글입니다. **길면 그만큼 자리를 차지해요.**

- **좋음:** 30~60줄
- **위험:** 200줄 넘어가면 오히려 AI가 규칙을 잘 안 지킵니다

**규칙: 매번 필요한 것만 넣습니다.** 한 번만 필요한 건 그냥 대화로 말하세요.

# 2. 슬래시 명령어 3개

Claude Code 대화창에서 `/` 를 치면 명령어 목록이 뜹니다. 이 중 3개만 알면 됩니다.

---

## `/context` — 얼마나 찼는지 본다

```
/context
```

컨텍스트 창이 몇 % 찼는지 보여줍니다. **돈 안 듭니다.**

**언제 치나:** 30분마다 한 번. 습관 들이세요.

**숫자 읽는 법**

| % | 상태 | 할 일 |
|---|---|---|
| ~30% | 좋음 | 계속 |
| 50% | **주의** | 곧 정리할 준비 |
| 70%+ | 나쁨 | `/compact` 또는 `/clear` |

> **50%가 기준입니다.** 100%가 아니라요. 절반 넘으면 이미 품질이 떨어지고 있습니다.

---

## `/compact` — 대화를 압축한다

```
/compact
```

지금까지의 대화를 **요약해서 줄입니다.** 흐름은 기억하되 부피를 줄여요.

**언제 쓰나:** 같은 작업을 계속할 건데 대화가 길어졌을 때

**요령: 뭘 남길지 알려주기**
```
/compact 지금 만들고 있는 리뷰 폼과 아까 발생한 에러를 중심으로 요약해줘
```

> **좋은 소식:** `/compact` 를 해도 `CLAUDE.md` 는 다시 읽힙니다. 규칙이 사라지지 않아요.

---

## `/clear` — 대화를 완전히 지운다

```
/clear
```

처음부터 다시 시작합니다. **컨텍스트 창이 100% 비워집니다.**

**언제 쓰나**
- **회차를 시작할 때** (매번!)
- 완전히 다른 작업으로 넘어갈 때
- AI가 너무 이상해졌을 때

**겁내지 마세요.** `CLAUDE.md` 와 만들어둔 파일은 그대로 남아 있습니다. 대화만 지워지는 거예요.

---

## `/compact` vs `/clear` 언제 뭘?

```
지금 하던 작업을 이어서 한다  →  /compact
새로운 작업으로 넘어간다      →  /clear
```

**예시**

| 상황 | 명령어 |
|---|---|
| 리뷰 폼 만들다가 대화가 길어짐. 계속 폼 작업할 거임 | `/compact` |
| 폼 다 만들었고 이제 색깔 다듬을 거임 | `/clear` |
| 오늘 회차 시작 | `/clear` |
| 에러 뚫다가 한 시간 헤맴. 뚫렸음 | `/clear` |

---

# 3. 회차 중 실천 규칙 4개

이것만 지켜도 사용량이 절반으로 줍니다.

### ① 회차 시작할 때 `/clear`

지난 회차 대화를 끌고 오지 마세요. `CLAUDE.md`가 다 기억하고 있습니다.

### ② 30분마다 `/context`

숫자만 확인하세요. 3초입니다.

### ③ 작업이 바뀌면 `/clear`

"폼 만들기" → "색깔 바꾸기" 는 다른 작업입니다.

### ④ 한 번에 하나만 시키기

이게 사실 제일 큰 절약입니다. 열 가지를 한 번에 시키면 AI가 파일을 잔뜩 읽고, 그게 다 컨텍스트에 쌓입니다.

> 이건 `core/02-prompting.md` 의 **원칙 1** 과 같은 이야기예요.
> **좋은 프롬프트가 곧 토큰 절약입니다.**

## 한도에 걸렸다면

회차 중에 한도가 차면 남은 시간을 못 씁니다. 그때는:

1. 옆 사람 화면을 보면서 따라갑니다
2. 집에서 한도가 풀리면 이어서 합니다
3. 카톡방에 알려주세요

> **예방이 최선입니다.** 위 4개 규칙만 지키면 2시간 회차에서 한도에 걸릴 일이 거의 없습니다.

# 5. 첫 세팅 체크리스트

1회차가 끝나고 집에서 10분만 투자하세요.

- [ ] 프로젝트 폴더에서 `/init` 실행
- [ ] 생성된 `CLAUDE.md` 를 위 템플릿대로 고치기
- [ ] `/context` 한 번 쳐보고 숫자 확인
- [ ] `/clear` 한 번 쳐보기 (겁내지 말고)
- [ ] `CLAUDE.md` 를 저장하고 닫기

이거 끝나면 6주 동안 다시 볼 일 거의 없습니다.

## 자주 나오는 질문

**Q. `/clear` 하면 지금까지 만든 게 사라지나요?**
아니요. **대화만** 사라집니다. 파일과 코드는 그대로예요. `CLAUDE.md` 도 그대로입니다.

**Q. CLAUDE.md 안 만들면 안 되나요?**
됩니다. 근데 매 회차마다 "한국어로 답해줘, 이건 Next.js야, 파일 고치기 전에 말해줘..." 를 다시 쳐야 해요. 그게 다 토큰입니다.

**Q. Codex 쓰는데요?**
`AGENTS.md` 라는 이름으로 같은 내용을 만드세요. 슬래시 명령어는 다를 수 있으니 `/help` 로 확인하세요.

**Q. 명령어가 문서랑 다르게 나와요.**
도구는 계속 업데이트됩니다. `/help` 를 치면 **지금 내 버전에 있는 명령어**가 나옵니다. 그게 정답입니다.
