## 시작하기 전에

이 커뮤니티에서 여러분이 진짜로 배우는 기술은 **코딩이 아니라 프롬프트**입니다.

AI는 코드를 씁니다. 여러분은 **뭘 만들지 정하고, 그걸 말로 전달합니다.**
6주 뒤에 남는 근육은 그거예요.

---

## 프롬프트 4원칙

### 원칙 1 · 한 번에 하나만

AI에게 열 가지를 한 번에 시키면, 여덟 개는 대충 하고 두 개는 틀립니다. 그리고 뭐가 틀렸는지 찾기가 어려워집니다.

**나쁜 예**
> 홈 화면 만들고 상세 페이지도 만들고 로그인 붙이고 색깔도 예쁘게 해줘

**좋은 예**
> 홈 화면을 만들어줘. 상단에 제목, 아래에 리뷰 목록 5개가 보이면 돼.

하나 끝나면 다음 걸 시킵니다. 이게 결과적으로 훨씬 빠릅니다.

---

### 원칙 2 · 결과를 묘사한다, 방법을 지시하지 않는다

우리는 코드를 모릅니다. 그러니 **"어떻게"를 시키면 안 됩니다.** 대신 **"뭐가 보였으면 좋겠는지"** 를 말합니다.

**나쁜 예**
> useState 훅으로 상태 관리해서 map으로 렌더링해줘

(모르는 말을 쓰면, 틀리게 씁니다)

**좋은 예**
> 리뷰를 입력하고 저장 버튼을 누르면, 아래 목록에 방금 쓴 리뷰가 바로 나타나면 좋겠어.

AI가 알아서 적절한 방법을 고릅니다.

---

### 원칙 3 · 구체적으로. 특히 숫자와 위치

**나쁜 예**
> 좀 예쁘게 만들어줘

**좋은 예**
> 배경은 흰색, 글씨는 진한 회색으로 해줘. 제목은 크게, 본문은 읽기 편한 크기로.

**나쁜 예**
> 버튼 추가해줘

**좋은 예**
> 리뷰 목록 위쪽 오른편에 "리뷰 쓰기" 버튼을 넣어줘.

---

### 원칙 4 · 안 되면 상황을 설명한다

이게 제일 중요합니다. **에러가 났을 때 "안 돼요"라고만 하면 AI도 모릅니다.**

**나쁜 예**
> 안 돼

**좋은 예**
> 저장 버튼을 눌렀는데 아무 일도 안 일어나. 화면도 안 바뀌고 에러 메시지도 안 보여.
> 터미널에는 이렇게 떠 있어:
> [에러 메시지 그대로 붙여넣기]

---

## 좋은 프롬프트 vs 나쁜 프롬프트

| 상황 | 이렇게 말고 | 이렇게 |
|---|---|---|
| 처음 만들 때 | "쇼핑몰 만들어줘" | "상품 목록을 보여주는 홈 화면 하나만 먼저 만들어줘" |
| 고칠 때 | "이거 이상해" | "제목이 화면 왼쪽에 붙어 있는데, 가운데로 옮겨줘" |
| 에러 났을 때 | "에러 나" | "터미널에 이 메시지가 떠. [붙여넣기] 뭘 확인해야 해?" |
| 이해 안 될 때 | (그냥 넘어감) | "방금 만든 파일 3개가 각각 무슨 역할인지 한 줄씩만 설명해줘" |
| 되돌리고 싶을 때 | "아까로 돌려줘" | "방금 바꾼 파일을 되돌려줘. 그 전 상태로." |

## 회차별 복붙 템플릿

**대괄호 [ ] 안을 자기 것으로 바꿔서** 그대로 복사해 쓰세요.

---

### 1회차 · 첫 프로젝트 만들기

```
Next.js 프로젝트를 새로 만들어줘.
TypeScript 없이, 가장 기본 설정으로.
폴더 이름은 [my-site] 로 해줘.
```

```
방금 만든 프로젝트를 실행해줘.
브라우저에서 볼 수 있게.
```

---

### 2회차 · 페이지 목록 뽑기

```
나는 [브런치 카페 리뷰를 모아두는] 사이트를 만들고 싶어.
쓰는 사람은 [나 혼자, 내가 다녀온 카페를 기록하려고].

첨부한 스크린샷은 참고하고 싶은 실제 사이트야.
이걸 참고해서, 내 사이트에 필요한 페이지 목록을 뽑아줘.
각 페이지에 뭐가 들어가는지도 같이 적어줘.

아직 코드는 만들지 마. 목록만.
```

목록이 나오면 이렇게 좁힙니다.

```
6주 안에 혼자 만들 거야. 지금 목록에서 꼭 필요한 것만 남기고 줄여줘.
페이지는 [3]개 이하로.
```

---

### 3회차 · 홈 화면 만들기

```
아까 정한 페이지 목록 중에서 홈 화면만 먼저 만들어줘.

홈 화면에 들어갈 것:
- [상단에 사이트 제목]
- [그 아래 리뷰 카드 목록]
- [카드에는 카페 이름, 별점, 한 줄 평]

데이터는 아직 없으니까 가짜 데이터 3개만 넣어줘.
```

```
지금 만든 걸 브라우저에서 볼 수 있게 실행해줘.
```

---

### 4회차 · 기능 붙이기

**먼저 폼만 (저장은 나중에)**
```
홈 화면에 [리뷰를 입력하는 폼]을 추가해줘.
입력 칸: [카페 이름], [별점], [한 줄 평]
저장 버튼을 누르면 아래 목록에 바로 나타나면 돼.

아직 데이터베이스는 연결하지 마. 화면에서만 되면 돼.
```

**그다음 Supabase 연결**
```
이제 Supabase에 연결해줘.

내 Supabase 정보야:
Project URL: [여기에 붙여넣기]
anon key: [여기에 붙여넣기]

테이블 이름은 [reviews] 이고, 칸은 [name, rating, comment] 야.

리뷰를 저장하면 Supabase에 들어가고,
새로고침해도 목록이 그대로 남아 있게 해줘.

키는 .env 파일에 넣어서 코드에 직접 안 보이게 해줘.
```

---

### 5회차 · 다듬기

```
지금 화면에서 이 부분을 고치고 싶어:
- [제목 글씨가 너무 작아. 두 배 정도 크게]
- [카드 사이 간격이 좁아. 조금 띄워줘]
- [배경이 흰색인데 아주 옅은 회색으로]

한 번에 하나씩 하고, 하고 나면 알려줘.
```

```
휴대폰 화면에서 봤을 때도 안 깨지게 해줘.
```

```
브라우저 탭에 뜨는 이름을 [내 카페 기록] 으로 바꿔줘.
```

**GitHub에 올리기**
```
이 프로젝트를 GitHub에 올려줘.
저장소 이름은 [my-site] 로, 비공개로 만들어줘.
.env 파일은 올라가지 않게 확인해줘.
```

---

### 6회차 · 배포

```
방금까지 작업한 걸 GitHub에 다시 올려줘.
```

배포 자체는 Vercel 화면에서 클릭으로 합니다. AI가 할 일이 아니에요.

**배포 후 사이트에 데이터가 안 나오면**
```
Vercel에 배포했는데 사이트에서 데이터가 안 나와.
로컬에서는 잘 됐어.
환경 변수 이름이 뭐로 되어 있어야 하는지 알려줘.
```

## 에러가 났을 때 (제일 중요한 장)

### 3단계로 하세요

```
1단계 · AI에게 에러 메시지를 그대로 붙여넣는다   (3분)
2단계 · 안 풀리면 카톡방에 스크린샷을 올린다      (즉시)
3단계 · 옥탑방에서는 옆 사람에게 물어본다
```

**혼자 30분 헤매지 마세요.** 그게 이 커뮤니티가 있는 이유입니다.

### 에러 프롬프트 템플릿

```
이 명령어를 실행했는데 에러가 났어:
[내가 친 명령어]

에러 메시지 전체야:
[빨간 글씨 전부 복사해서 붙여넣기]

뭐가 문제인지, 뭘 확인해야 하는지 알려줘.
```

### 에러 메시지를 복사하는 법

터미널에서 빨간 글씨가 좌르르 뜨면, **처음 5줄만** 복사해도 대부분 충분합니다.
드래그해서 `Ctrl + C` 로 복사됩니다.

### 같은 에러가 반복될 때

```
같은 에러가 세 번째야. 아까 알려준 방법대로 했는데 또 났어.
지금 상태를 다시 확인해줘. 뭘 놓쳤을까?
```

## AI가 이상한 걸 만들었을 때

### 되돌리기

```
방금 바꾼 걸 전부 되돌려줘. 그 전 상태로.
```

### 왜 그렇게 했는지 묻기

```
왜 그 파일을 새로 만들었어? 한 줄로만 설명해줘.
```

### 너무 많이 바꿨을 때

```
파일을 너무 많이 바꿨어.
앞으로는 파일을 고치기 전에 뭘 바꿀지 한 줄로 먼저 말해줘.
```

이 마지막 문장은 아예 `CLAUDE.md` 에 넣어두면 매번 안 말해도 됩니다.
→ `core/03-token-setup.md` 참고

## 이해 안 될 때 쓰는 프롬프트

넘어가도 되지만, 가끔 물어보면 실력이 늡니다.

```
방금 만든 파일들이 각각 무슨 역할인지 한 줄씩만 설명해줘.
전문 용어는 빼고.
```

```
지금 이 명령어가 뭘 하는 건지 초등학생한테 설명하듯 알려줘.
```

```
이 코드에서 내가 나중에 고치고 싶어질 만한 부분이 어디야?
```

## 하지 말아야 할 것

| | 왜 |
|---|---|
| **API 키를 채팅창에 그대로 노출** | Supabase 키는 `.env`에. AI에게 줄 때도 "이 키를 .env에 넣어줘" 라고 함께 말하기 |
| **"알아서 다 해줘"** | AI가 뭘 만들지 모릅니다. 여러분도 뭐가 만들어졌는지 모르게 됩니다 |
| **에러를 스스로 고치려고 코드 직접 수정** | 우리는 코드를 모릅니다. 더 큰 에러가 납니다. AI에게 상황을 설명하세요 |
| **모르는 채로 계속 `y` 누르기** | 최소한 "지금 뭐 하려는 거야?" 한 번은 물어보세요 |
