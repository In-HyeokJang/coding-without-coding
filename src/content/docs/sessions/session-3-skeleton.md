## 오늘 미리 알아두면 좋은 것

**오늘 처음으로 눈에 보이는 게 나옵니다.**

지금까지 폴더랑 문서만 있었죠. 오늘은 브라우저를 열면 **내가 만든 화면**이 뜹니다.

그리고 그 순간을 **6명이 다 같이 봅니다.** 각자 준비해두고, 카운트다운하고, 동시에 새로고침.

---

- [ ] **참조 사이트 스크린샷**
- [ ] 프로젝트 폴더가 있는 노트북

**6일 동안 아무것도 못 했어도 괜찮습니다. 그냥 오세요.**

## 오늘 새로 설치하는 것

**없습니다.** 다만 오늘 처음 쓰는 명령어가 하나 있습니다.

### `npm run dev` — 내 컴퓨터에서 사이트 켜기

내 노트북 안에 **작은 웹서버**를 띄우는 명령어입니다.

실행하면 터미널에 이렇게 뜹니다.
```
ready in 1200ms
- Local: http://localhost:3000
```

그다음 브라우저 주소창에 `localhost:3000` 을 치면 내 사이트가 뜹니다.

> **중요:** 이 주소는 **나만 볼 수 있습니다.** 옆 사람 컴퓨터에서는 안 열려요.
> 세상에 공개하는 건 6회차입니다.

---

## 오늘 하는 것 3단계

### 1단계 · AI에게 홈 화면 만들라고 시키기 (30분)

**프롬프트**
```
아까 정한 페이지 목록 중에서 홈 화면만 먼저 만들어줘.

홈 화면에 들어갈 것:
- 상단에 사이트 제목
- 그 아래 리뷰 카드 목록
- 카드에는 카페 이름, 별점, 한 줄 평

데이터는 아직 없으니까 가짜 데이터 3개만 넣어줘.
```

> **팁:** `CLAUDE.md` 에 페이지 목록을 넣어뒀다면 "아까 정한 목록대로" 만 해도 알아듣습니다.

파일이 여러 개 생깁니다. 다 이해하려고 하지 마세요.

---

### 2단계 · 서버 켜기 (30분)

```
지금 만든 걸 브라우저에서 볼 수 있게 실행해줘.
```

또는 직접:
```
npm run dev
```

**여기서 에러가 많이 납니다. 정상입니다.**

터미널에 초록색으로 `ready` 가 뜨면 성공입니다.

> **이 순간 잠깐 멈춰보세요.** 검은 화면이 지금 내 지시대로 움직이고 있습니다.
> 6주 전엔 이 화면 앞에서 손이 멈췄던 사람도 있어요.

---

### 3단계 · 다 같이 3, 2, 1 (10분)

6명 다 브라우저에 `localhost:3000` 을 쳐놓고 **엔터 치기 직전 상태**로 맞춥니다.

먼저 된 사람은 못 된 사람을 도와주세요. **다 같이 맞춥니다.**

그리고 카운트다운.

```
3 ... 2 ... 1 ...
```

**각자 자기 화면과 처음 만납니다.**

## 오늘 손에 남는 것

**내 컴퓨터에서 뜨는 홈 화면 하나.**

아직 클릭해도 아무 반응 없습니다. 예쁘기만 해요.
그래도 **6주 전엔 없던 겁니다.**

## 자주 막히는 곳

| 증상 | 프롬프트 |
|---|---|
| `npm run dev` 하니까 에러가 좌르르 | 에러 메시지를 그대로 AI에게 붙여넣기 |
| 서버는 떴는데 브라우저에 아무것도 안 나옴 | "화면이 하얗게 비어 있어. 뭘 확인해야 해?" |
| 포트 3000이 이미 쓰이고 있대요 | "3000번 포트가 이미 사용 중이래. 어떻게 해?" |
| AI가 파일을 너무 많이 만들었어요 | 괜찮습니다. 다 필요한 겁니다 |

**에러 프롬프트 템플릿**
```
이 명령어를 실행했는데 에러가 났어:
npm run dev

에러 메시지야:
[빨간 글씨 처음 5줄 복사해서 붙여넣기]

뭐가 문제인지, 뭘 확인해야 하는지 알려줘.
```
