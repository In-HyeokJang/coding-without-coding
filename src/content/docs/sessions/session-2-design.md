> **오늘의 목표:** 내 사이트에 어떤 페이지가 필요하고, 각 페이지에 뭐가 들어갈지 정하기
> **오늘 손에 남는 것:** 페이지 목록 문서 한 장

---

## 오늘 미리 알아두면 좋은 것

어제 폴더는 만들었는데, **뭘 만들어야 할지는 아직 모릅니다.**

"카페 리뷰 사이트요" 정도로는 AI가 화면을 못 만들어요.
**"홈에는 이거, 상세에는 이거"** 까지 정해야 3회차에 화면이 뜹니다.

오늘 그걸 정합니다. **코드는 안 씁니다.**

---

## 오기 전에

- [ ] 어제 만든 프로젝트 폴더가 있는 노트북
- [ ] **참조 사이트 하나 떠올려 오기** (필수 X, 있으면 시간 절약)
- [ ] 메모할 곳 (노션, 메모장, 아무거나)

**참조 사이트가 뭔가요?**
내가 만들고 싶은 것과 비슷한 **실제로 있는 사이트** 하나.

> 예: 브런치 카페 리뷰 사이트를 만들고 싶다 → 실제 맛집 리뷰 사이트 하나

똑같이 만들려는 게 아닙니다. **"여기 이 부분이 나한테 필요해요"** 를 짚기 위해서예요.

---

## 오늘 새로 설치하는 것

**없습니다.** 어제 세팅한 걸 그대로 씁니다.

---

## 오늘 하는 것 3단계

### 1단계 · 참조 사이트 뜯어보기 (25분)

1. 참조하고 싶은 사이트를 브라우저에 엽니다
2. 화면을 스크린샷으로 저장합니다
3. Claude Code에 스크린샷을 주고 대화를 시작합니다

**프롬프트**
```
나는 [브런치 카페 리뷰를 모아두는] 사이트를 만들고 싶어.
쓰는 사람은 [나 혼자, 내가 다녀온 카페를 기록하려고].

첨부한 스크린샷은 참고하고 싶은 실제 사이트야.
이걸 참고해서, 내 사이트에 필요한 페이지 목록을 뽑아줘.
각 페이지에 뭐가 들어가는지도 같이 적어줘.

아직 코드는 만들지 마. 목록만.
```

---

### 2단계 · AI와 대화하며 좁히기 (25분)

AI가 목록을 뽑아주면, 그대로 받지 마세요. **너무 많을 겁니다.**

```
6주 안에 혼자 만들 거야.
지금 목록에서 꼭 필요한 것만 남기고 줄여줘.
페이지는 3개 이하로.
```

```
[상세 페이지]에 뭐가 들어가야 하는지 더 구체적으로 알려줘.
```

이렇게 20분 넘게 대화하세요. **오늘 AI와 오래 이야기하는 게 목적입니다.**

---

### 3단계 · 옆 사람에게 설명하기 (30분)

가장 중요한 시간입니다.

2인 1조로 앉아서, 자기 페이지 목록을 **말로 설명합니다.**

> "제 사이트는 홈이랑 상세, 두 페이지예요. 홈에는 카드가 쭉 있고..."

듣는 사람은 **자기 언어로 되물어보세요.**
> "이 페이지가 이런 뜻이에요?"
> "여기 이거 왜 넣었어요?"

**여기서 뭔가 일어납니다.** 설명하다가 자기 입으로 "어? 이 페이지 왜 넣었지?" 하게 돼요.
그 순간이 오늘의 핵심입니다.

---

### 마지막 · 목록 확정

옆 사람과 얘기하다 발견한 걸 다시 AI에게 던져서 고칩니다.

```
방금 다시 생각해봤는데 [검색 페이지]는 빼는 게 나을 것 같아.
목록을 다시 정리해줘.
```

**확정된 목록을 파일로 저장하세요.** 노션이든 메모장이든.

---

## 오늘 손에 남는 것

**페이지 목록 문서.** 이런 모양입니다.

```
홈 페이지
  - 상단에 사이트 제목
  - 아래에 리뷰 카드 목록
  - 카드에는 카페 이름, 별점, 한 줄 평
  - 우측 상단에 "리뷰 쓰기" 버튼

상세 페이지
  - 리뷰 전문
  - 별점
  - 작성 날짜
```

**이게 있어야 3회차에서 진짜 화면을 만들 수 있습니다.**

---

## 집에 가서 5분만

확정한 페이지 목록을 `CLAUDE.md` 의 `## 페이지 구조` 자리에 붙여넣으세요.

그러면 다음 회차부터 AI가 **매번 물어보지 않아도 내 사이트 구조를 압니다.**

> `CLAUDE.md` 가 뭔지 모르시면 → `core/03-token-setup.md`

- **옆 사람에게 설명할 때 방해받지 않고 3분**
- **듣는 사람은 자기 언어로 되묻기**
