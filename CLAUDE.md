# 코딩 없이 코딩 · 문서 사이트

## 이 프로젝트가 뭔가

광명시 청년 대상 바이브코딩 커뮤니티 **"코딩 없이 코딩"** 의 참가자용 가이드 문서 사이트.
Astro Starlight로 만들고 GitHub Pages에 배포한다.

**읽는 사람:** 코딩을 모르는 30대 초반 직장인. 폰으로 읽는 경우가 많다.

## 스택

- Astro + Starlight
- GitHub Pages 배포 (GitHub Actions)
- 패키지 매니저: npm

## 명령어

- 개발 서버: `npm run dev`
- 빌드: `npm run build`
- 빌드 미리보기: `npm run preview`

## 폴더 구조

```
src/content/docs/     문서 본문 (md/mdx)
  index.mdx           홈
  core/               도구, 프롬프트, 토큰 설정
  sessions/           1~6회차
astro.config.mjs      Starlight 설정 (사이드바 여기)
.github/workflows/    Pages 배포
```

## 작업 규칙

- 한국어로 답한다.
- 설명은 짧게. 세 줄을 넘기지 않는다.
- 파일을 고치기 전에 무엇을 왜 바꾸는지 한 줄로 먼저 말한다.
- 한 번에 파일 하나씩만 고친다.
- 새 패키지를 설치하기 전에 반드시 물어본다.
- 요청하지 않은 파일은 건드리지 않는다.

## 문서를 쓸 때 지킬 것

- **원본 md의 문장을 함부로 바꾸지 않는다.** 구조와 형식만 Starlight에 맞게 옮긴다.
- 참가자는 코딩을 모른다. 전문 용어를 쓰면 괄호 안에 쉬운 말로 풀어쓴다.
- 회차 문서는 "오늘의 목표 → 오기 전에 → 오늘 하는 것 → 손에 남는 것 → 다음 회차" 순서를 지킨다.
- 프롬프트 예시는 반드시 코드블록에 넣는다. 참가자가 복사해서 쓴다.

## 시각화 원칙

- **다이어그램은 4개까지만.** 도구 흐름도, 6회차 진행 곡선, 4회차 도달 3갈래, 결과물 타임라인.
- 그 외에는 표나 글로 충분하다. 다이어그램을 늘리자고 제안하지 않는다.
- Claude Code / Codex 처럼 두 갈래로 갈리는 안내는 탭으로 만든다.
- 경고나 주의는 Starlight의 aside 문법을 쓴다.

## 하지 말 것

- 원본 md에 없는 내용을 지어내지 않는다. 필요하면 물어본다.
- 설치 명령어나 버전 번호를 기억에 의존해 쓰지 않는다. 확실하지 않으면 "확인 필요" 라고 표시한다.
- 디자인을 화려하게 만들지 않는다. 읽기 쉬운 게 먼저다.
